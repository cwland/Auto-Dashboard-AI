Welcome to Auto Dashboard AI

Turn bookmarks into an AI-organized new-tab dashboard. Setup takes about two minutes.

1

Get an AI Provider API Key

Pick an AI provider in Settings (OpenAI, Anthropic, Google, Groq, and more) and paste its API key. OpenRouter is the easiest place to start — one key gives you access to dozens of models, with free-tier credits that are more than enough to generate several dashboards.

Open OpenRouter →
💡 Create a free account, then click Create Key. Copy the key — you'll paste it in the next step.
2

Enter & Validate Your Key

Click the Settings tab above, paste your API key into the field, and click Validate. Once validation succeeds, click Save Settings.

🔒 Your API key is stored locally on your device only. It is never sent to any server other than OpenRouter.
3

Choose an AI Model

In Settings, pick a model from the dropdown. The default Gemini Flash 1.5 is fast, accurate, and inexpensive. If you'd rather pay nothing, choose one of the Free models.

⚡ Gemini Flash 1.5 typically processes 50 bookmarks in under 10 seconds.
4

Create Your First Dashboard

Open the Dashboards tab. Use the bookmark tree on the left to select folders or individual bookmarks. Give your dashboard a name, then click ✨ Generate Dashboard. The AI will add descriptions and resolve icons automatically.

🎉 After generating, open a new tab — your dashboard will be waiting for you!

Privacy & Security

This extension runs entirely in your browser. It has no backend servers and collects no analytics — it only talks to the services you set up.

No data collection

Auto Dashboard AI has no servers of its own and sends no telemetry or analytics anywhere. Your bookmarks, dashboards, and usage stay on your device. The only outbound connections are the ones you explicitly configure: your chosen AI provider, the services your widgets connect to, and — only if you turn it on — your own GitHub Gist for backup.

Where your settings & API keys are stored

All settings, dashboards, and credentials — AI provider API keys, widget/integration tokens, your GitHub token, and your backup passphrase — are stored locally in your browser (chrome.storage.local) on this device. They are never uploaded anywhere except as described in “Backups” below, and the GitHub token and passphrase are never uploaded at all.

AI provider & data retention

When you use an AI feature (organizing bookmarks into sections, generating theme palettes), the data needed for that request — bookmark titles, URLs and descriptions, or your theme prompt — is sent directly from your browser to the AI provider you selected (e.g. OpenRouter, OpenAI, Anthropic, Google, Groq), along with your API key for that provider.

What the provider logs or retains is governed by that provider's own data-retention and privacy policy, not by this app. Please review your provider's terms. The app sends only what's required for the action you triggered and keeps no copies on any server. If you never use the AI features, no bookmark data is sent to any provider.

Backups & encryption

Backup is optional and off by default. If you enable Gist backup, your settings and dashboards are uploaded to a private gist on your own GitHub account.

  • Encrypted before it leaves your browser. Backups are encrypted client-side with AES-256-GCM, using a key derived from your passphrase via PBKDF2-SHA256. A passphrase is required to turn on sync, so the synced contents are always encrypted. (Only a tiny version/schema header is left unencrypted.)
  • Your token and passphrase are never uploaded. The GitHub token and the backup passphrase stay on your device only — they are stripped from the backup payload. So even if the gist were exposed, the data is encrypted and your credentials aren't in it.
  • Your other API keys are inside the backup. Integration and AI keys are part of your settings, so they're included within the encrypted backup. Anyone who has both your gist contents and your passphrase could read them — keep the passphrase private, and share it only with your own devices.
  • Use a strong, unique passphrase and keep your gist private. You can also use Export/Import to move an (encrypted) backup file manually.

Widget connections — HTTPS vs HTTP

Live widgets connect from your browser directly to the service URLs you enter (Proxmox, Tautulli, etc.), using the credentials you provide. The connection's safety depends on the URL scheme:

  • HTTPS (recommended): the entire connection — including API tokens and passwords — is encrypted in transit. Always prefer it.
  • HTTP (risky): the connection is unencrypted. Your API tokens and data travel in clear text and can be read by anyone on the network path (other devices on your LAN, routers, your ISP). Only use HTTP on a network you fully trust, and switch to HTTPS whenever the service supports it.
  • Self-signed HTTPS certificates: some self-hosted services use them — you may need to open the service's URL once in your browser and accept the certificate. The connection is still encrypted.

Tokens are sent only to the host you configured for that widget, and only when the widget refreshes.

How to Use

Pick a task below — each is a short, step-by-step walkthrough.

🗂️ Build a dashboard
1

Open the Dashboards tab and start a new one

Click the 🗂️ Dashboards tab at the top, then + Create Dashboard. A short wizard opens.

2

Name it and pick a type

Give the dashboard a Name (the only required field). Optionally set a description, Canvas size, and Layout mode. Choose 🧩 Bookmarks & Widgets to fill it from your bookmarks, or 📄 Blank Dashboard to start empty. Click Next.

3

Choose how bookmarks are organized

Pick 📁 Folder structure (mirrors your bookmark folders, no AI needed), ✨ Dynamic folder structure (AI adds icons & descriptions), or ✨ Dynamic AI categorization (AI regroups by topic). Select the folders or individual bookmarks from the tree — or click Skip Adding Bookmarks for a blank board.

💡 The AI options only appear once an AI provider is set up under Settings → 🤖 AI Provider.
4

Generate it

Click ✨ Generate Dashboard. When it finishes, open a new tab to see it live.

5

Search and switch

On the dashboard, press / to jump to Search bookmarks… and filter every section as you type; press Esc to clear. With two or more dashboards you can switch using the tabs, sidebar, or dropdown (choose the style under Settings → ⚙️ General Settings). The default dashboard — marked with a ★ — is the one that opens on new tabs and at startup.

📡 Add a live widget
1

Turn on the integration

Open the 🔧 Integrations tab, find the service you want (e.g. Proxmox, Tautulli, Weather, Uptime Kuma), and flip its toggle on.

2

Enter and validate the connection

Fill in the service's URL and API key/token, then click its Validate / Test Connection button. Once it succeeds, use Preview Widget to confirm live data appears.

🔒 Prefer an https:// URL — with plain http your token and data travel unencrypted over the network.
3

Enter edit mode on a dashboard

Open a dashboard in a new tab and click ✎ Edit Dashboard in the top bar. A small toolbar appears.

4

Add the widget

Click ➕ in the toolbar, choose Add Widget (“Live data from a configured integration”), pick your widget, and click Add to board.

5

Place it and save

Drag the widget where you want it and resize it on the grid, then click ✓ to save your layout. (✕ discards changes.)

🎨 Set a theme
1

Open the Theme settings

Go to the Settings tab and choose the 🎨 Theme sub-tab.

2

Pick Light, Dark, or Custom

Use the selector at the top to switch between the Light, Dark, and Custom palette groups.

3

Click a palette

Click any swatch to apply it instantly as a live preview — the settings page and your dashboard update right away.

4

Or build your own

On the Custom tab, click + Add custom theme to pick each color by hand, ✨ Generate with AI to describe a vibe, or 🎲 Surprise me to let the AI invent one. Pick a generated palette and Add selected.

5

Save

Click Save Settings to keep your choice.

💾 Back up & sync
1

Open the Backup settings

Go to the Settings tab and choose the 💾 Backup sub-tab.

2

Quick local backup (optional)

Click Export configuration to download an encrypted backup file you can keep anywhere; use Import configuration… to restore it later. This needs no account.

3

Turn on cloud sync

To sync across computers, switch on Back up to a private GitHub Gist. Your data is encrypted on this device before it's uploaded, so GitHub never sees your settings or keys.

4

Add a passphrase and token

Enter an Encryption passphrase (required) and paste a GitHub access token, then click Test to confirm the token works.

5

Back up — and sync automatically

Click Back up to Gist to push your first copy. On another computer, click Restore from Gist. Turn on Keep in sync automatically for hands-off two-way sync.

🔑 Your GitHub token and passphrase never leave this device. The passphrase is the only key to your backup — store it somewhere safe, because it can't be recovered.

💾 Backup & Restore

Save a copy of all your settings and dashboards to a file you can keep or move to another computer. Import a file to restore everything (this replaces what you have now). The file includes your saved API keys, so keep it somewhere private.

Save an encrypted copy of your dashboards and settings to a private GitHub Gist, so you can restore it on any computer or browser. Your data is encrypted on this device before it's uploaded, so GitHub never sees your settings or keys. Turn this on, then add a passphrase and a GitHub token below.

A password used to scramble your backup before it's uploaded. Use the same passphrase on every computer you want to sync. ⚠️ Keep it safe — if you forget it, the backup can't be recovered.

This lets the extension save your backup to a private Gist on your GitHub account. Create one with Gist access in your GitHub token settings, paste it here, and click Test to check it.

Two-way sync across computers running the same browser (Brave ↔ Brave, Chrome ↔ Chrome — each brand keeps its own backup). When on, your changes are backed up automatically (about 30 seconds after you stop editing), and newer changes from your other computers are loaded automatically (on startup, every 30 seconds, and when you open a dashboard). The most recent change wins. To copy between different browsers, use Export/Import. You can still back up or restore by hand with the buttons above.
